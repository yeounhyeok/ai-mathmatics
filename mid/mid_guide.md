🔧 NumPy 함수 체크리스트(plot 제외, ipynb 코드 셀 기준)

시험에 나올 가능성이 높은 함수/연산
• np.array([...])
  - 리스트를 NumPy 배열로 만든다.
  - shape 확인과 행렬 계산의 시작점.
  - 예: x = np.array([1, 2, 3]) -> shape (3,)
  - 예: X = np.array([[1, 2], [3, 4]]) -> shape (2, 2)

• @, np.dot(a, b), np.matmul(A, B)
  - @ : 행렬곱/벡터 내적에 가장 자주 사용.
  - np.dot(a, b): 1D 벡터끼리는 내적.
  - np.matmul(A, B): 행렬곱 함수. np.linalg.matmul 이 아니라 np.matmul 이 맞음.
  - 내부 차원이 맞아야 한다.
  - 예: (m, n) @ (n,) -> (m,)
  - 예: (m, n) @ (n, 1) -> (m, 1)

• .T
  - 2D 배열에서는 전치.
  - 1D 배열에서는 shape가 바뀌지 않는다.
  - 예: np.array([1,2,3]).T 의 shape는 여전히 (3,)

• reshape(...)
  - 배열의 shape를 바꾼다.
  - 예: x.reshape(-1, 1) -> column vector 형태.
  - 예: x.reshape(1, -1) -> row vector 형태.
  - -1은 남은 차원을 자동 계산하라는 의미.

• np.newaxis
  - 새로운 축을 추가한다.
  - 예: x[:, np.newaxis] -> (n,)을 (n, 1)로 바꿈.
  - broadcasting이나 column vector 만들 때 중요.

• np.ones(shape), np.zeros(shape), np.zeros_like(x)
  - np.ones((m, 1)): bias column 만들 때 자주 사용.
  - np.zeros(n): 초기 가중치나 빈 벡터 만들 때 사용 가능.
  - np.zeros_like(x): x와 같은 shape의 0 배열.
  - shape는 튜플로 쓰는 습관: np.ones((3, 1)).

• np.hstack([...]), np.concatenate([...]), np.column_stack([...])
  - np.hstack: 배열을 가로 방향으로 붙임. bias trick에서 중요.
  - 예: X_bias = np.hstack([np.ones((m, 1)), X])
  - np.concatenate: 배열을 이어 붙이는 일반 함수. axis 지정에 주의.
  - np.column_stack: 1D 배열들을 열로 세워 2D 행렬을 만들 때 사용.

• np.mean(x, axis=...), np.std(x, axis=...), np.sum(...)
  - mean/std: 표준화에서 핵심.
  - axis=0: 열별 계산.
  - axis=1: 행별 계산.
  - 표준화는 보통 훈련 데이터 기준으로 axis=0.
  - 예: mu = np.mean(X_train, axis=0)
  - 예: std = np.std(X_train, axis=0)
  - 예: X_scaled = (X - mu) / std
  - np.sum은 MSE, gradient에서 자주 사용.

• np.linspace(a, b, c)
  - a부터 b까지 c개의 값을 만든다.
  - a와 b가 모두 포함된다.
  - 간격 개수는 c-1개.
  - 예: np.linspace(0, 1, 5) -> [0, 0.25, 0.5, 0.75, 1]

• np.arange(start, stop, step)
  - start부터 stop 전까지 step 간격으로 만든다.
  - stop은 포함되지 않는다.
  - linspace와 차이 주의.

• np.exp(x), np.log(x)
  - np.exp: sigmoid에서 e^x 계산.
  - np.log: log-likelihood에서 사용.
  - 예: sigmoid = 1 / (1 + np.exp(-z))
  - 예: log_likelihood = k*np.log(p) + (n-k)*np.log(1-p)

• np.linalg.inv(A), np.linalg.norm(x)
  - np.linalg.inv(A): 역행렬. 정규방정식 w* = (X.T X)^(-1) X.T y 에서 등장.
  - np.linalg.norm(x): 벡터 크기, weight decay 직관과 연결 가능.
  - 역행렬은 계산 비용이 크고 항상 존재하지 않을 수 있음.

• np.linalg 주요 함수
  - np.linalg은 선형대수(linear algebra) 함수들이 모여 있는 모듈.
  - np.linalg.inv(A): 정사각행렬 A의 역행렬.
  - np.linalg.norm(x): 벡터/행렬의 크기. 벡터 길이, weight 크기와 연결.
  - np.linalg.solve(A, b): Ax = b를 직접 푸는 함수. 역행렬을 직접 구하는 것보다 보통 더 안정적.
  - np.linalg.det(A): 행렬식. det(A)=0이면 역행렬이 없음.
  - np.linalg.eig(A): 고유값/고유벡터. 시험 우선순위는 낮음.
  - 주의: 행렬곱 함수는 np.linalg.matmul이 아니라 np.matmul 또는 @.

알아두면 좋은 보조 함수
• np.allclose(a, b)
  - 두 계산 결과가 거의 같은지 확인.
  - 시험 빈칸 가능성은 낮지만 실습 코드에서 자주 보임.

• np.random.seed(...), np.random.uniform(...), np.random.randn(...)
  - 데이터 생성용.
  - 시험에서는 직접 구현보다 의미 확인 정도.

• np.stack([...]), np.c_[...], ravel()
  - 배열을 쌓거나 펼치는 함수.
  - 시각화/격자 데이터 생성에서 많이 보이므로 우선순위 낮음.
  - plot 관련 문제는 제외.

함수 관련 최우선 암기
• shape를 먼저 본다.
• 1D transpose는 안 바뀐다.
• @는 행렬곱이고, broadcasting은 +, -, 요소별 곱/나눗셈에서 주로 발생한다.
• bias trick은 np.ones((m, 1))와 np.hstack을 같이 쓴다.
• 표준화는 np.mean/std axis=0을 훈련 데이터 기준으로 계산한다.
• np.linspace는 끝값 포함, np.arange는 stop 미포함.

## 3. NumPy 배열과 shape

### 3-1. shape를 항상 확인해야 함
노트에서 가장 강하게 반복되는 포인트:

> **NumPy 구현 시 shape를 항상 확인하는 습관이 필요하다.**

### 3-2. 1차원 배열
- 1D 배열은 그냥 숫자 묶음들
- shape: `(n,)`
- 행이 따로 있는 게 아님
- 노트에는 **column vector 느낌으로 이해**하려는 흔적이 있음
  - 다만 NumPy의 1차원 배열은 엄밀히는 행벡터/열벡터가 명시되지 않은 상태라고 보는 게 안전

### 3-3. 2차원 배열
- 예: `[[1]]`
- 이것도 행렬임
- shape: `(1, 1)`

### 3-4. 차원 판단 감각
- **NumPy에서는 괄호 중첩 개수가 차원과 연결된다**
- shape를 안 보고 감으로만 계산하면 실수하기 쉬움

### 3-5. 전치(Transpose)
- **1차원 배열은 transpose해도 안 바뀜**
- 이유: 1차원 배열은 애초에 `(원소개수,)` 형태라서, 행/열 구분이 분명하지 않기 때문

### 3-6. 자주 헷갈리는 포인트
- 일부 지정 → **인덱싱(indexing)**
- 덩어리째 끌어오기 → **슬라이싱(slicing)**

---

## 4. 벡터/행렬 연산

### 4-1. 내적 (Dot product)
- 내적은 이 과목에서 기본적인 벡터 연산의 핵심
- 벡터끼리의 관계, 선형모델 계산의 기초가 됨

### 4-2. `@` 연산자
- `@` : 행렬곱셈
- 노트 포인트:
  - **앳 연산 = 행렬곱셈**
  - 다만 1차원 배열 다룰 때는 직관이 깨질 수 있으니 shape 주의

### 4-3. 자주 확인할 것
- `2D @ 1D -> 1D`
- `2차원 @ 1차원 배열 -> 1차원 배열 shape`

즉,
- 행렬 × 벡터를 하면 결과가 다시 벡터(1D)로 나오는 경우가 많다
- 계산 후에도 **shape를 바로 확인**해야 한다

### 4-4. 결론
행렬곱은 그냥 숫자 계산이 아니라,
**입력 shape와 출력 shape를 같이 추적하는 연산**이다.

## 7. Broadcasting
이 과목에서 가장 중요하게 반복되는 개념 중 하나.

### 7-1. 직관적 이해
노트 표현을 살리면:
- **브로드캐스팅:**
  - "야, 너 왜 나보다 작아? 내 크기에 맞춰서 니 몸집을 불려(복사해)!"
  - 주로 덧셈/뺄셈/요소별 곱에서 발생
- **행렬곱(@):**
  - "내 가로랑 네 세로 숫자 맞지? 그럼 우리 규칙대로 합치자."
  - 자동 축 맞춤과는 다르게, 행렬곱 규칙을 따름

### 7-2. 브로드캐스팅 규칙
- 짧은 쪽의 **왼쪽을 1로 채움**
- 둘 중 하나가 **1이면**, 상대 크기에 맞게 복사 가능
- 둘 다 2 이상인데 다르면 복사 어려움

### 7-3. 자주 나오는 해석
- **행렬 A에 벡터 B의 전치행렬을 더하는 것처럼 보이는 상황**
- 결국은 **각 행에 동일한 벡터를 더한다**는 의미로 이해 가능

노트식 표현:
- “똑같은 걸 다 알아서 더하니 방송(Broadcasting)”

### 7-4. 브로드캐스팅과 정규화 연결
- Z-score normalization에서 broadcasting이 적극 활용됨
- 평균 벡터, 표준편차 벡터를 전체 데이터에 한 번에 적용할 수 있게 해줌

## 8. 정규화와 표준화

### 8-1. Z-score Normalization
노트에서 강조한 내용:
- broadcasting의 활용 사례
- 큰 값들을 그대로 다루면 문제가 발생함
- 큰 값에 작은 값이 묻혀서 특징이 드러나지 않을 수 있음
- 면적이 너무 커지는 문제를 해결
- 학습이 불안정해지는 문제를 완화

즉,
**표준화는 값의 스케일을 맞춰서 학습을 더 안정적으로 만드는 과정**으로 이해하면 된다.

### 8-2. normalize vs standardization
노트에 직접 언급됨:
- 정규화(normalize)
- 표준화(standardization)

둘을 구분해서 볼 필요가 있다.

- **정규화(normalize)**: 범위를 맞추는 느낌
- **표준화(standardization)**: 평균 0, 표준편차 1 기준으로 바꾸는 느낌

이 과목 노트에서는 특히 **Z-score normalization(표준화)** 가 핵심으로 보인다.

## 9. 선형성 / 선형변환

### 9-1. 선형성
노트 핵심:
- 행렬의 열벡터로 각각 나눈 후,
- 기저벡터가 그 열벡터로 이동한다고 생각
- 기저벡터가 정해지면 나머지 점들도 자동 결정됨

즉,
**행렬은 공간 전체를 마음대로 뒤트는 게 아니라, 기저벡터의 이동 방식을 정하는 도구**라고 이해하면 된다.

### 9-2. 선형변환
- 기저벡터의 변환을 곱하여 공간 전체의 벡터를 변환함
- 각 벡터의 상관관계는 유지됨

핵심은:
- 선형변환은 공간의 구조를 완전히 부수는 게 아니라,
- **선형 관계를 보존하면서 변환**하는 것

### 9-3. 편향(bias) 포함 모델 해석
노트에 나온 중요한 문장:
- **편향을 포함한 모델도 순수한 선형 변환의 수학으로 분석할 수 있게 된다**

즉,
편향항까지 포함해도 적절한 차원 확장/트릭을 쓰면 선형대수 틀 안에서 볼 수 있다.

## 11. 넘파이 논리연산

### 배열 비교/논리연산 주의
- `np.array` 비교 연산에서 `and`가 아니라 **`&`** 사용

### 논리연산 기호
- and : `&`
- or : `|`
- not : `~`

즉 Python의 일반 boolean 논리연산과 NumPy 배열 논리연산은 구분해야 한다.

## 12. 선형공간 / 데이터 생성

### `np.linspace(a, b, c)`
- a부터 b까지 c개의 점을 가진 선형공간 생성
- 노트 표현:
  - “a부터 b까지 5개 걸친 선형공간”
  - 간격은 `c-1`로 나눠야 전체 c개가 나옴

### `np.random.uniform(lo, hi, n)`
- `[lo, hi)` 범위에서 n개의 실수 데이터 랜덤 생성

### 차이
- `linspace`: 일정 간격 데이터
- `uniform`: 랜덤 실수 데이터

## 13. 손실(loss) 계산 관련 주의
노트에서 꽤 중요한 실수 방지 포인트가 하나 있음.

### 핵심 문장
- **손실을 행렬에 넣을 때 잔차를 계산하자마자 넣어라**

### 의미
현재 모델 상태와 기록된 손실이 서로 엇갈리면 안 된다.

노트 해석:
- 현재 모델 상태 `w_new`
- 기록된 손실 상태 `Loss_old`
- 이 둘이 서로 짝이 안 맞는 상태가 되면 안 됨

즉,
**모델 파라미터를 업데이트한 뒤의 상태와, 손실 기록 시점이 어긋나면 해석이 꼬인다.**

## 16. 암기용 한 줄 요약
- **shape 안 보면 거의 틀린다.**
- **1D transpose는 안 바뀐다.**
- **`@`는 행렬곱, broadcasting은 덧셈/뺄셈/요소곱 쪽 자동 확장이다.**
- **브로드캐스팅은 작은 배열을 큰 배열 규격에 맞춰 복사해 계산하는 개념이다.**
- **Z-score normalization은 스케일 문제를 줄여 학습 안정성을 높인다.**
- **선형변환은 기저벡터가 어디로 가는지를 정하는 것이라고 보면 편하다.**
- **NumPy 배열 논리연산은 `&`, `|`, `~`를 쓴다.**
