## 상세 피드백

### 1번: 1D 배열과 2D 열벡터

정답:

```text
v.shape = (3,)
v.T.shape = (3,)
v_col.shape = (3,1)
W @ v -> (4,)
W @ v_col -> (4,1)
```

네 답은 shape는 전부 맞다.

다만 설명은 이렇게 쓰는 게 더 정확하다.

```text
v는 1D 배열이라 행벡터/열벡터 구분이 없어서 v.T를 해도 shape가 (3,) 그대로이다.
v_col은 2D 열벡터 shape (3,1)이므로 전치하면 shape가 (1,3)이 된다.
```

`@` 결과가 다른 이유와 `.T` 결과가 다른 이유는 연결되어 있지만, 시험에서는 `.T` 질문이면 전치 자체를 먼저 설명하는 게 좋다.

### 2번: 행렬곱과 Broadcasting

정답:

```text
A.shape = (3,2)
b.shape = (2,)
c.shape = (3,)

(a) A + b -> 가능, 결과 (3,2)
(b) A + c -> 불가능
(c) A @ b -> 가능, 결과 (3,)
(d) A @ c -> 불가능
```

가장 중요한 오답:

```text
(3,2) + (3,) 은 불가능
```

이유:

```text
(3,2)
  (3,)
```

뒤 차원부터 비교하면 `2`와 `3`이 맞지 않고, 둘 중 하나가 1도 아니므로 broadcasting 불가능이다.

정확한 문장:

```text
Broadcasting은 주로 덧셈, 뺄셈, 요소별 곱/나눗셈에서 작은 배열을 큰 배열에 맞춰 확장하는 규칙이고,
행렬곱 @는 앞 행렬의 열 수와 뒤 배열의 행 방향 차원이 맞아야 가능한 연산이다.
```

### 3번: 인덱싱 방식 구분

전부 좋다.

정답:

```text
X[2, 1]       -> 기본 인덱싱, scalar, shape ()
X[1:3, 1:]    -> slicing, shape (2,2)
X[X > 50]     -> boolean indexing, shape (4,)
X[[0, 3]]     -> fancy indexing, shape (2,3)
X[:, [2, 0]]  -> fancy indexing, shape (4,2)
```

`X[X > 50]` 설명도 맞다. Boolean mask로 참인 원소만 뽑으면 원래 2D 위치 구조가 보존되지 않고, 선택된 원소들이 1D로 나열된다.

### 4번: Z-score 정규화와 axis

axis 답은 맞다.

```text
특성별 평균 -> axis=0
샘플별 평균 -> axis=1
```

다만 (c), (d)는 더 정확히 이렇게 쓰는 게 좋다.

```text
Z-score 정규화는 각 특성의 평균을 0, 표준편차를 1로 맞춰 특성 간 스케일 차이를 줄이기 위해 사용한다.
```

```text
X.mean(axis=1)을 쓰면 샘플별 평균을 빼게 되므로 특성별 스케일을 맞추지 못한다.
또 shape가 (3,)이어서 X의 shape (3,2)와 바로 broadcasting도 되지 않는다.
```

마지막 설명은 방향이 좋다. 단, 표현은 이렇게 다듬으면 A+ 답안이다.

```text
특성 스케일 차이가 크면 MSE 손실 등고선이 길쭉한 타원이 되어 gradient descent가 지그재그로 움직이거나 느리게 수렴한다.
Z-score 정규화는 특성들의 스케일을 맞춰 손실 등고선을 원에 가깝게 만들고, 수렴을 더 안정적이고 빠르게 만든다.
```

주의:

```text
경사가 원모양이 되는 것이 아니라, 손실 등고선이 원에 가까워진다.
```

### 5번: 코드 빈칸과 개념 설명

전부 좋다.

정답:

```python
mean = X_raw.mean(axis=0)
std = X_raw.std(axis=0)
X_norm = (X_raw - mean) / std

ones = np.ones((3, 1))
X_aug = np.hstack([X_norm, ones])

y_hat = X_aug @ w
```

shape:

```text
X_norm -> (3,2)
ones -> (3,1)
X_aug -> (3,3)
y_hat -> (3,1)
```

`np.ones(3)`과 `np.ones((3,1))` 차이도 잘 설명했다.

## 바로 외울 Broadcasting 기준

뒤 차원부터 비교한다.

```text
같으면 가능
둘 중 하나가 1이면 가능
둘 다 1이 아니고 서로 다르면 불가능
```

예시:

```text
(3,2) + (2,)   -> 가능, (3,2)
(3,2) + (3,)   -> 불가능
(3,2) + (3,1) -> 가능, (3,2)
(2,1) + (1,3) -> 가능, (2,3)
```
