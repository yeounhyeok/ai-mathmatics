## 핵심 오답: A-9

문제:

```text
u: (5,)
v: (5,1)

u + v = ?
```

정답:

```text
가능, 결과 shape = (5,5)
```

이유:

```text
u.shape = (5,)    -> broadcasting 비교에서는 (1,5)처럼 생각
v.shape = (5,1)

   (1,5)
+  (5,1)
--------
   (5,5)
```

뒤 차원부터 비교:

```text
5 vs 1 -> 가능, 1이 5로 확장
1 vs 5 -> 가능, 1이 5로 확장
```

그래서 결과는 `(5,5)`이다.

## 현재 상태

Lecture 1의 약점 보완 상태는 많이 좋아졌다.

```text
1D vs 2D shape: 95%
transpose/reshape: 90-95%
axis=0/1: 95%
표준화 코드: 95%
broadcasting: 85-90%
```

이제 Lecture 1은 A+권에 가깝다. 남은 건 broadcasting에서 `1D 배열이 앞쪽에 1을 보충한 것처럼 해석된다`는 규칙을 자동화하는 것이다.

## 마지막 암기

```text
(5,) + (5,1)
```

이건 불가능이 아니라:

```text
(1,5) + (5,1) -> (5,5)
```

1D 배열은 broadcasting 비교 시 필요한 만큼 **왼쪽에 1이 붙는 것처럼** 생각한다.
