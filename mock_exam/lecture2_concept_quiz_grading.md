## 상세 피드백

### 1번. 행렬의 열과 기저벡터

정답:

```text
A @ e1 = [2, 3]
A @ e2 = [-1, 4]
```

이 값들은 각각 A의 첫 번째 열벡터, 두 번째 열벡터이다.

`x = [5, 2]`일 때:

```text
A @ x = 5 * [2, 3] + 2 * [-1, 4]
      = [10, 15] + [-2, 8]
      = [8, 23]
```

네 답은 선형결합 표현까지 맞다. 최종 벡터 `[8, 23]`까지 쓰면 더 좋다.

추가 설명은 이렇게 쓰면 A+ 답안이다.

```text
모든 벡터 x는 기저벡터 e1, e2의 선형결합으로 표현된다.
행렬 A는 e1, e2가 어디로 이동하는지를 열벡터로 저장한다.
선형변환은 선형결합을 보존하므로, 기저벡터의 목적지를 알면 공간의 모든 벡터가 어디로 이동하는지 알 수 있다.
```

### 2번. 선형변환 조건과 편향 트릭

판단은 모두 맞다.

```text
T1(x) = A @ x      -> 선형변환
T2(x) = A @ x + b  -> 일반적으로 선형변환 아님
T3(x) = 2x         -> 선형변환
T4(x) = x + 3      -> 선형변환 아님
```

편향 트릭 설명은 방향은 맞지만, 더 정확히는 다음과 같다.

```text
A @ x + b는 원래 공간에서는 선형변환이 아니라 affine 변환이다.
하지만 입력에 1을 추가해 [x1, x2, 1]처럼 차원을 확장하면,
b를 가중치의 마지막 성분으로 넣을 수 있어 하나의 행렬곱으로 표현할 수 있다.
즉 원래 공간의 affine 변환을 확장된 공간의 선형 연산처럼 다루는 것이다.
```

핵심 표현:

```text
원래 공간에서는 선형변환이 아니지만, 확장된 공간에서는 행렬곱으로 표현 가능하다.
```

### 3번. Broadcasting 규칙과 의미

정답:

```text
M: (3,2)
r: (2,)
c: (3,1)
d: (3,)

(a) M + r -> 가능, (3,2)
(b) M + c -> 가능, (3,2)
(c) M + d -> 불가능
(d) r + c -> 가능, (3,2)
```

네 오답:

```text
r + c = X
```

정확한 해석:

```text
r.shape = (2,)   -> (1,2)처럼 생각
c.shape = (3,1)

  (1,2)
+ (3,1)
-------
  (3,2)
```

뒤 차원부터 비교하면:

```text
2 vs 1 -> 가능
1 vs 3 -> 가능
```

따라서 가능하고 결과는 `(3,2)`이다.

반복 방향 설명은 좋다.

```text
M + r: r이 행마다 세로 방향으로 반복된다.
M + c: c가 열 방향, 즉 가로로 반복된다.
```

### 4번. `np.newaxis`, `reshape`

전부 정답이다.

```text
x.shape -> (3,)
x[:, np.newaxis].shape -> (3,1)
x[np.newaxis, :].shape -> (1,3)
x.reshape(-1,1).shape -> (3,1)
x.reshape(1,-1).shape -> (1,3)
```

추가 연산:

```text
x[:, np.newaxis] + np.array([1, 2])

(3,1) + (2,) -> (3,1) + (1,2) -> (3,2)
```

이것도 맞았다.

설명은 이렇게 다듬으면 더 정확하다.

```text
x[:, np.newaxis]는 shape (3,1)인 열벡터를 만든다.
x[np.newaxis, :]는 shape (1,3)인 행벡터를 만든다.
```

### 5번. 편향 트릭과 코드 빈칸

거의 정답이다.

```text
y_hat = 3*x1 - 2*x2 + 7
w_aug = [[3], [-2], [7]]
```

```text
X.shape = (4,2)
X_aug.shape = (4,3)
```

코드:

```python
m = X.shape[0]
ones = np.ones((m, 1))
X_aug = np.hstack([X, ones])
w_aug = np.array([[3], [-2], [7]])
y_hat = X_aug @ w_aug
```

네 답의 `np.ones((4,1))`도 이 문제에서는 맞다.  
다만 코드 빈칸에서는 이미 `m = X.shape[0]`가 주어졌으므로 `np.ones((m,1))`이 더 일반적이고 좋은 답이다.

개념 설명도 좋다. 특히 MSE 미분과 이후 계산에서 깔끔해진다는 설명은 괜찮다.
