## 주요 오답 분석

### 1. 1D와 2D 행렬곱 shape

`(3,) @ (3,1)`은 가능하다. 결과는 `(1,)`이다.

헷갈린 이유는 `(3,)`이 행벡터처럼 동작하긴 하지만, NumPy에서는 엄밀히 1D array라서 결과도 2D `(1,1)`이 아니라 1D `(1,)`로 나온다.

시험 직전 기억:

```text
(3,) @ (3,)   -> scalar
(3,) @ (3,1) -> (1,)
(1,3) @ (3,) -> (1,)
(1,3) @ (3,1)-> (1,1)
```

### 2. Fancy indexing shape

`X[[0,2], [1,2]]`는 두 위치를 각각 뽑는다.

```text
X[0,1], X[2,2]
```

그래서 결과는 행렬 `(2,2)`가 아니라 벡터 `(2,)`이다.

반대로 `X[:, [2,0]]`는 모든 행에서 2번, 0번 열을 뽑으므로 `(4,2)`가 맞다.

### 3. 정규방정식의 행렬 순서

정답은 다음 순서다.

```text
w* = (X.T @ X)^(-1) @ X.T @ y
```

네 답은 의도는 맞았지만 `y @ X.T`처럼 보이는 순서가 들어갔다. 행렬곱은 교환법칙이 성립하지 않으므로 이건 실전에서 감점이 크다.

체크 포인트:

```text
X.T @ y 는 가능
y @ X.T 는 보통 shape가 안 맞거나 의미가 다름
```

### 4. Sigmoid chain rule

`d/dw ln sigma(wx)`는 아래처럼 한 번만 `x`가 곱해진다.

```text
d/dw ln sigma(wx)
= sigma'(wx) / sigma(wx) * x
= sigma(wx)(1 - sigma(wx)) / sigma(wx) * x
= x(1 - sigma(wx))
```

네 풀이에서는 마지막에 `x`가 한 번 더 곱해져 `x^2(1 - sigma(wx))`처럼 되었다. 이건 개념 부족보다는 chain rule 정리 중복 실수다.

### 5. MLE 이계도함수

이항분포 log-likelihood에서 이계도함수는 최대점 확인을 위해 음수여야 한다.

```text
ell''(p) = -k / p^2 - (n-k) / (1-p)^2
```

두 항이 모두 음수라서 `p = k/n`에서 최대값이 된다. 부호 하나가 바뀌면 최대/최소 판단이 흔들린다.
