## 1. 1D/2D Shape 행렬곱 10문제

각 연산이 가능한지 O/X로 표시하고, 가능하면 결과 shape를 쓰시오.

```text
1.  a: (3,),    b: (3,)      -> a @ b
2.  a: (3,),    B: (3, 1)    -> a @ B
3.  A: (1, 3),  b: (3,)      -> A @ b
4.  A: (1, 3),  B: (3, 1)    -> A @ B
5.  A: (4, 3),  b: (3,)      -> A @ b
6.  A: (4, 3),  B: (3, 1)    -> A @ B
7.  a: (4,),    B: (4, 3)    -> a @ B
8.  A: (4, 1),  b: (4,)      -> A @ b
9.  A: (2, 3),  B: (3, 4)    -> A @ B
10. A: (2, 3),  b: (2,)      -> A @ b
```

## 1.1. `@` 연산과 Broadcasting 혼합 12문제

각 연산이 가능한지 O/X로 표시하고, 가능하면 결과 shape를 쓰시오.

```text
1.  A: (4, 3),  b: (3,)      -> A @ b + 1
2.  A: (4, 3),  b: (3, 1)    -> A @ b + 1
3.  A: (4, 3),  b: (3,)      -> A @ b + np.ones((4,))
4.  A: (4, 3),  b: (3,)      -> A @ b + np.ones((4, 1))
5.  A: (4, 3),  b: (3, 1)    -> A @ b + np.ones((4,))
6.  A: (4, 3),  b: (3, 1)    -> A @ b + np.ones((4, 1))
7.  x: (3,),    y: (3,)      -> x @ y + x
8.  x: (3,),    B: (3, 2)    -> x @ B + np.ones((2,))
9.  x: (3,),    B: (3, 2)    -> x @ B + np.ones((2, 1))
10. A: (2, 3),  B: (3, 1)    -> A @ B + np.ones((2,))
11. A: (2, 3),  B: (3, 1)    -> A @ B + np.ones((2, 1))
12. A: (2, 3),  b: (3,)      -> A @ b + np.ones((1, 2))
```

---

## 1.2. `@` 결과 후 Broadcasting 재훈련 12문제

각 연산이 가능한지 O/X로 표시하고, 가능하면 결과 shape를 쓰시오.

```text
1.  x: (5,),    y: (5,)      -> x @ y + np.ones((5,))
2.  x: (5,),    y: (5,)      -> x @ y + np.ones((5, 1))
3.  A: (3, 5),  x: (5,)      -> A @ x + np.ones((3,))
4.  A: (3, 5),  x: (5,)      -> A @ x + np.ones((3, 1))
5.  A: (3, 5),  x: (5, 1)    -> A @ x + np.ones((3,))
6.  A: (3, 5),  x: (5, 1)    -> A @ x + np.ones((1, 3))
7.  x: (3,),    B: (3, 4)    -> x @ B + np.ones((4, 1))
8.  x: (3,),    B: (3, 4)    -> x @ B + np.ones((1, 4))
9.  A: (2, 3),  B: (3, 2)    -> A @ B + np.ones((2,))
10. A: (2, 3),  B: (3, 2)    -> A @ B + np.ones((2, 1))
11. A: (2, 3),  b: (3,)      -> A @ b + np.ones((2, 1))
12. A: (2, 3),  b: (3,)      -> A @ b + np.ones((3,))
```

## 1.3. 최종 검증: `@` + Broadcasting 종합 15문제

각 연산이 가능한지 O/X로 표시하고, 가능하면 최종 결과 shape를 쓰시오.  
중간에 `@` 결과 shape를 먼저 옆에 적고 판단하시오.

```text
1.  a: (4,),    b: (4,)      -> a @ b + np.ones((1,))
2.  a: (4,),    B: (4, 1)    -> a @ B + np.ones((1, 1))
3.  A: (1, 4),  b: (4,)      -> A @ b + np.ones((1, 1))
4.  A: (1, 4),  B: (4, 1)    -> A @ B + np.ones((1,))
5.  A: (3, 4),  b: (4,)      -> A @ b + np.ones((3, 1))
6.  A: (3, 4),  B: (4, 1)    -> A @ B + np.ones((3,))
7.  A: (3, 4),  B: (4, 2)    -> A @ B + np.ones((2,))
8.  A: (3, 4),  B: (4, 2)    -> A @ B + np.ones((3, 1))
9.  x: (3,),    M: (3, 4)    -> x @ M + np.ones((1, 4))
10. x: (3,),    M: (3, 4)    -> x @ M + np.ones((4, 1))
11. M: (3, 4),  x: (4,)      -> M @ x + np.ones((1, 3))
12. M: (3, 4),  x: (4,)      -> M @ x + np.ones((3, 1))
13. A: (2, 3),  b: (3,)      -> (A @ b).reshape(2, 1) + np.ones((2,))
14. A: (2, 3),  b: (3,)      -> (A @ b).reshape(1, 2) + np.ones((2, 1))
15. A: (2, 3),  b: (2,)      -> A @ b + np.ones((2,))
```

## 2. Fancy Indexing Shape 5문제

다음 배열을 기준으로 각 코드의 결과 shape를 쓰고, 어떤 방식의 인덱싱인지 쓰시오.

```python
X = np.array([
    [10, 20, 30, 40],
    [50, 60, 70, 80],
    [11, 22, 33, 44],
    [55, 66, 77, 88],
    [ 1,  2,  3,  4]
])
```

```python
1. X[[0, 2, 4]]
2. X[:, [3, 1]]
3. X[[0, 2], [1, 3]]
4. X[[0, 2]][:, [1, 3]]
5. X[X[:, 0] > 20][:, [0, 2]]
```
